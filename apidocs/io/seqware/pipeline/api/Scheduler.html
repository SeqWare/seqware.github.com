        This method just needs a sw_accession value from the workflow table and an ini file(s) in order to schedule a workflow. All needed info is pulled from the workflow table which was populated when the workflow was installed. Keep in mind this does not actually trigger anything, it just schedules the workflow to run by adding to the workflow_run table. This lets you run workflows on a different host from where this command line tool is run but requires an external process to launch workflows that have been scheduled.
